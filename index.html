写在荒野大镖客2：救赎之后（未写完

发表于 2021-11-19

今天是2021年11月19日，距离2018年的神仙大战已经三年（距离黑暗降临还有一年），战神4拿到了TGA、世界成为了影响力最大最广的怪猎、奥德赛变成了新时代的刺客信条、蔚蓝依旧是最牛逼的独立游戏之一，然而我要说的主角是那一年最让人难忘的red dead redemption 2。时至今日，大可以抛开所有的热度和话题，仅从一个玩家的角度，为R星给世界玩家留下的这部3A时代最后的艺术品，献上发自灵魂的赞叹。
因为我相信，无论时光如何流转，次时代怎样发展，red dead redemption 2都将永远在所有玩家的心中留下浓墨重彩的一笔

第九艺术与游戏

在中国，对于电子游戏，有着一个极失偏颇的代称：“第九艺术”。然而在中文互联网这个极容易陷入魔怔之中的环境里，这个名字被强行塞给一切有点热度的游戏，无论它们是否配得上这份殊荣。
当然我不是想对第九艺术的魔怔进行口诛笔伐，只不过这部荒野大镖客2正可以说是第九艺术其代名词。

半路杀出与强弩之末

冬日，遮蔽天空的暴风雪，不到5米的能见度和不知时间的恶劣天气，十几个人与几辆马车，这是rdr2的开头，这是再普通不过的开头，没有“多年以后奥雷里亚诺上校面对行刑队”的设置悬念，也没有“这是一个最好的时代，也是一个最坏的时代”的总结，总之就是，你，我，暂时不知道是谁，我们被困在了天气恶劣的雪山之上。
于是我们需要找个地方休整一下，梳理一下现在的情况，想办法弄点物资补给。这时候一个八字胡的牛仔骑着马跑了过来，你的角色上来就和他骂了起来，你这时当然还不认识这个游戏史上最最最让人痛恨的恶棍，也不认识这个穿着笔挺讲究的红色西装高高在上发号施令的混蛋，总之就是去农庄化个缘。一路上你知道你们是个土匪帮派，和敌对帮派打了一场枪战后还救了女人出来，天这么黑，你满脑子想的就是一件事：赶紧把这帮npc安顿好，然后去这个游戏史上最逼真的雪地里畅快体验19世纪美国的冬日西部风景。
可是第二天起来，一个女人跑过来让你救救她老公，你当然不认识她，也不认识她老公，包括游戏里这个蓝衣服牛仔也骂骂咧咧的，但是你还是会说，why not？毕竟是任务，有钱有事儿做还能顺路看风景。
于是rdr2最巧妙的叙事方法就此将你拖入了这个牛仔将亡的新时代美国，一发不可收拾。

作为叙事顺序上的半路上车，游戏并没有一开始就清楚交代帮派有谁、如何成立、为什么逃亡，而是让玩家在早期任务中和一个个帮派成员单独出行，在聊天中一点一点交代故事背景和人物塑造，从忠心耿耿的墨西哥小胡子哈维尔，到年轻莽撞的憨憨约翰，到足智多谋的老将何西阿，charming的义贼头头达奇，人狠话不多却善良诚实的黑人印第安混血查尔斯，脑子灵活身手却不行的开心果西恩，年轻老实任劳任怨的黑人小伙兰尼，还有四肢发达头脑简单的比尔，和骁勇善战的女武神莎迪，以及不折不扣的恶棍麦卡。当然还有你自己，这个和谐可爱的大家族中的一员，大家都信赖的big man及神枪手，亚瑟·摩根。
当然我们后来都知道，和谐温暖不过只是表象，实际上在文明逐渐遍布这片美洲大陆的19世纪末，这样的土匪帮派早就是强弩之末了，范德林德帮也不过是从文明人的城市被赶出来的贼。如果我们在打完后回过头来分析，范德林德帮的崩溃可能并不始于罗兹镇的失利，甚至应该追溯到黑水镇劫案开始，也就是游戏开始的时间线之前。在游戏开始的时候，这个属于cowboys和outlaws的时代就已经迎来末路了。
可是这种半路上车的叙事顺序却暂时性的掩盖了强弩之末的表象，玩家像从头开始处一个新朋友一样，和这帮陌生又可爱的人度过一天又一天，去抢劫一个个从人民身上敲骨吸髓的资本家，随着性子帮助路边的陌生人，或是出自正义感惩治那些到处作恶的敌对帮派，又或是顺着土匪本性抢劫普通平民。

总之，过去的事情你一概不知，未来的发生自有天注定，很快你就融入了这个世界，你就成为了亚瑟摩根。

Spring框架的学习笔记

发表于 2021-05-05 更新于 2021-10-03

Spring

Spring的核心是IoC（控制反转）和DI（依赖注入）。

面向对象开发中，由于对象之间的继承关系，对象A的运行需要一个依赖对象B，这使得软件耦合度很高。
IoC是一种第三方容器，它使得对象A和对象B之间不再有直接关系，当对象A需要依赖对象B运行时，IoC容器会自动给对象A注入依赖对象B，达到解耦的目的。

IoC的技术实现，依靠的是反射（映射）机制。

在Spring中，用户所写的类以Java Bean的形式在pom.xml文件中配置并交由Spring统一管理

Spring MVC

使用了MVC三层结构（Controller、Model、View）作为软件设计理念的Spring框架，主要用于写后端代码。

对前端写出来的.html格式的静态页面和css、图片等静态资源进行管理和调配，并在其中注入Java对象的技术。

Spring Boot

对ssm框架即Spring、SpringMVC、MyBatis的整合管理。简化开发步骤，将springMVC中传统的xml配置改为java配置，如java bean的配置不再需要xml文件，而是改为@bean注解。
AOP: 面向切面编程。理解为将不同业务中相同的重复事务交给框架统一管理，实现代码高复用。

Spring Cloud

正如其名，是一个云端的SpringBoot，可以允许业务由独立的模块实现，并将业务整合起来。
支持各模块的高度独立，实现去中心化。

Servlet和JSP的比较

Servlet：是Java程序，是javax.servlet包中的一个接口，主要包含5个方法，其中常用的有三个：
void init() 初次创建servlet对象时执行该方法，执行初始化。
void service() 用于向客户端发出响应的方法，每次客户端发来一次请求都会执行一次。
void destroy() 结束servlet的生命周期。
使用servlet时需要写一个自定义类继承servlet接口并完善需要的功能。
JSP：用于在HTML界面中插入Java代码，本质上是一个Servlet。
相较于同样可以插入在HTML标签中的JS而言，更容易和服务端进行交互。
语法为：

<% Java语句 %>

直接插入在HTML标签之间
JSP内置了9个对象：
request：表示一次客户端的请求 常用
response：表示一次服务端的响应 常用
pageContext：页面信息 常用
session：表示一次会话 常用
application：表示当前的 web 应用（是一个全局对象，保存所有用户的共享信息 常用
config：当前JSP对应的Servlet的ServletConfig对象，用于获取当前JSP对应的Servlet信息
out：用于向浏览器页面输出，JspWritter
page：当前JSP对应的Servlet对象
exception：表示JSP页面的异常

request常用方法

String getParameter(String key)
获取客户端传来的参数
void setAttribute(String key, Object value)
存储数据，注意是使用Object存储的
Object getAttribute(String key)
通过关键字取出数据
RequestDispatcher getRequestDispatcher(String path)
返回一个RequestDispatcher对象，该对象的forward方法用于请求的转发（区别与response的重定向，转发的请求和原请求是同一请求）
String[] getParameterValues()
获取客户端传来的多个同名参数
void setCharacterEncoding(String charset)
指定请求的编码

response常用方法

sendRedirect(String path)
重定向，页面间的跳转（区别与request的转发，重定向是由客户端重新发起一次新的请求）

Session

用于存储一次会话过程。
会话指从客户端从访问服务端到关闭浏览器之间的一系列交互行为。会话中可以产生多次请求和响应，属于同一个会话的请求都有同一个sessionID。
实现会话的方式有两种：
1.Session
2.Cookie

Session常用方法

String getID()
获取sessionID
void setMaxInactiveInterval(int interval)
设置session的失效时间，单位为秒
int getMaxInactiveInterval()
获取当前session的失效时间
void invalidate()
设置session立即失效
void setAttribute(String key, Object value)
存储数据
Object get Attribute(String key)
读取数据
void removeAttribute(String key)
删除数据

Cookie

是HTTP响应中传给浏览器的一个小文本文件，浏览器保存了某个Cookie后，在之后的请求和响应中，会将此Cookie来回传递，通过Cookie这个载体实现服务端与客户端的来回交互。
